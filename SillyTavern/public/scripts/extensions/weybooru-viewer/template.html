Search Syntax Guide

By default, search is for tags. You can search for any combination of tags seperated by spaces, and specified search terms must be matched for a post to be shown.

Alternatives:a,b,c (instead of spaces) matches if any of a, b, or c are found.

Exclusion: Prefix a tag with - to exclude posts with that tag from search results.

Named Filters: You can use the name:value syntax to filter posts by specific attributes, including:

  • id:123 - post number
  • score:5.. - score at least 5 (ranges: 1..4, ..10, 1..)
  • type:image - image, animation, flash, or video
  • uploader:name - filter by uploader name
  • safety:safe - filter by safety level
  • pool:1 - filter by pool ID
  • fav-count:1.. - filter by favorite count
  • special:fav - filter to posts you've favorited
  • special:liked - filter to posts you've liked

Min & Max: Numeric keys also support score-min:1 and score-max:10 style bounds.

Dates: Values such as today, yesterday, or YYYY-MM-DD can be used with date fields (e.g. creation-date:2024-01-01..).

Escaping: Put \ before : or - inside a tag name (example: re\:zero).
